# Sample Milestone Plan

This is a small example for a fictional tool. Replace it with a plan generated from your own `intake/problem-intake.md`. The milestones are then created by the agent in Github along with associated issues that define the work that needs to be done.

## v0.1.0 - Project Skeleton

Goal: Create the initial project structure, setup instructions, and synthetic data policy.

Issues:

- Add basic project structure.
- Add synthetic data fixtures.
- Document local setup.

Acceptance criteria:

- A beginner can clone the repo, read the quickstart, and run the first local command.

## v0.2.0 - First Workflow

Goal: Let a user process one synthetic record and produce a simple output.

Issues:

- Load synthetic records.
- Validate required fields.
- Produce a human-readable summary.

Acceptance criteria:

- The workflow runs using only synthetic data.

## v1.0.0 - First Major Release

Goal: Deliver the first approved version that satisfies the project definition of success.

Issues:

- Complete final synthetic test pass.
- Confirm documentation is accurate.
- Create the first GitHub release.

Acceptance criteria:

- The tool satisfies the stated mission goal using synthetic test scenarios.
